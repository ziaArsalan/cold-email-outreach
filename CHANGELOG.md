### 2026-07-07 — [T-015] Follow-up sequences — multi-step campaigns with stop-on-reply
- **Added:** Campaigns can now be multi-step sequences. `Campaign.steps: [{ order, templateId, delayDays }]` (empty = single-touch, backward-compatible via `templateId`); `QueuedEmail.stepIndex`. **Enqueue-at-send-time:** `start()` enqueues only step 0; when the worker marks a step-N item `sent` it schedules step N+1 for that lead at `sentAt + delayDays * config.followupDelayUnitMs` (renders the step's template with the lead's cached `ai_intro`; the next-step enqueue runs *before* the campaign-completion count so a mid-sequence campaign never wrongly completes). **Stop-on-reply/bounce/unsubscribe:** the worker checks the lead's status before every send — `replied`/`bounced`/`unsubscribed` → item flips to `cancelled` (new `queueService.markCancelled`), never sent, no further steps scheduled. Deliverability gate at start now validates EVERY step's template (a bad follow-up blocks the whole start, naming the step). New `config.followupDelayUnitMs` (`FOLLOWUP_DELAY_UNIT_MS`, default 1 day; shrunk in tests). Client: New Campaign form gets a follow-up sequence builder (+ Add follow-up rows: template + wait-N-days + remove), campaign list shows step count, live queue shows a Step column. `campaignService` exports `normalizeSteps` + `enqueueStepForLead`. New `server/scripts/testSequences.js`.
- **Area:** both
- **QA:** PASS — browser + DB-verified with QA leads on the user's own aliases (worker on, `FOLLOWUP_DELAY_UNIT_MS=3000`): 3-step campaign progressed step0→step1→step2 (follow-ups scheduled at sentAt+delay, not sent early); marking a lead replied flipped its pending step-2 to `cancelled` ("stopped — lead replied") with no step 3; a no-steps campaign sent exactly once (zero follow-up rows) and auto-completed; a 2-link template as step 2 rejected the start with 400 "Step 2 template fails deliverability rules" and zero queue rows. In-process `SEQUENCES PASS` + regressions `QUEUE`/`ROTATION`/`DELIVERABILITY`/`EMAIL VERIFICATION` all PASS; clean CRA build. All QA artifacts removed; 45 real leads intact; QA emails only to `zia20isys+seq*` aliases.
- **Commit:** T-015

### 2026-07-06 — [T-010] Email queue + scheduler worker
